This post is a visual check for the blog's callout component. The names below are grouped by color, but each one has its own default label and icon.

## Informational callouts

!!! note
General context that is useful but not urgent.

!!! abstract
A short description of a longer section.

!!! summary
The useful parts collected in one place.

!!! tldr
The shortest possible version.

!!! info
Neutral information with its own icon.

!!! todo
Work that still needs doing.

!!! question
A question for the reader or author.

!!! help
Guidance for someone who is stuck.

!!! faq
A frequently asked question.

!!! example
A concrete example of the idea above it.

## Positive callouts

!!! tip
A practical shortcut or better way to do something.

!!! hint
A smaller nudge in the right direction.

!!! important
Something worth emphasizing without raising an alarm.

!!! success
An operation completed successfully.

!!! check
Something has been verified.

!!! done
The work is finished.

## Caution callouts

!!! warning
Ignoring this advice could cause a recoverable problem.

!!! attention
This part needs a closer look.

!!! caution
Slow down before continuing.

## Destructive callouts

!!! failure
An operation did not complete.

!!! missing
Something expected could not be found.

!!! fail
The shorter failure label.

!!! danger
An action could cause data loss or downtime.

!!! error
The system returned an error.

!!! bug
Known behavior that needs fixing.

## Quote callouts

!!! quote
A highlighted quotation or attribution.

!!! cite
A source or reference worth separating from the body.

## Content variations

!!! info ""
This callout intentionally has no title or icon row.

!!! note "A custom title"
Titles can be replaced when the default label is too vague. Bodies accept **emphasis**, `inline code`, and [links](/projects/).

!!! tip "Lists work too"

    - Keep the title short.
    - Let longer content wrap naturally.
